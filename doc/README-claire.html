5) And finally, here's a problem Mike encountered - this is the kind of thing we'll need to troubleshoot; I'm sure Andrew will be able to help here:


The main problem seems to be that the system was built with an old version of Grails (1.2.0 according to Andrew), which comes with dozens of external code libraries in the form of JAR files, each of which sits in a particular directory (some sharing certain directories, others elsewhere). I have now tried at least five different versions of Grails (including three different releases of 1.2.0, called 1.2.0, 1.2.0-RC1 and 1.2.0-RC2), and none of them seems to have the right set of libraries in the right places, according to what the existing code wants. To give just one example, the project is looking for GRAILS_HOME/ant/lib/ant.jar within the Grails directory, whereas the current version of Grails has the equivalent file in GRAILS_HOME/lib/ant-1.7.1.jar. Different versions of Grails find this file, but miss others. While I can find this probably-correct-file-with-different-name-and-location for some of the files it wants, I cannot find it for others. There is also a "grails upgrade" command that comes with the current version of Grails (1.3.1) which is supposed to update any Grails project built with an old version so that it works with the new version. This did not work either; there were still plenty of JAR files the project couldn't find. My best guess as to what's going on here is that an early version of Grails was initially used, with some other libraries added/moved manually in various places.
