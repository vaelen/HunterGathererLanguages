NOTE: This is a message from Andrew Young about how to get the Grails app running.  One thing not mentioned here that needs to be done is obtain the DataSource.groovy file, which contains the production database password, and put it in the grails-app/conf directory.


WARNING WARNING WARNING: do *NOT* try to run grails without a valid DataSource.groovy file -- it will overwrite many source files in bad ways.


Mike,


I was able to get the HunterGathererLanguages program working with the version of groovy and grails that I emailed you the link to without having to install anything else and without any versioning problems.  Here are the steps I went through.  Notes are underlined, commands I ran are in bold.  The commands are also listed at the bottom of the email in an easy-to-copy format.


The commands below will start the app in development mode, using an embedded Jetty server and an embedded HSQL database.  


First we configure our groovy and grails environment

Andrew-Youngs-MacBook-Pro:~ vaelen$ cd hgl

Andrew-Youngs-MacBook-Pro:hgl vaelen$ curl -O http://vaelen.org/hgl/groovy-grails.tar.gz
